初阶使用

如何将资源复制进Kindle 3

只需要将充电插头从线束上拔下来，然后用那条USB线连接电脑USB和Kindle 3就可以权将Kindle 3当作一只U盘使用了。在电脑新盘符下有.active-content-data、audible、documents、music和system五个文件夹。

.active-content-data：空的，删除后连接USB又会自动生成，可能相当于一个缓存目录，未确认。 audible：有声书的文件夹。 documents：图书目录，MOBI，PRC，TXT，PDF等文件都放于其中，不论是否在其下再建新目录。 music：MP3目录，好象只支持MP3格式吧，你不得不佩服，Kindle 3的音质非常好。 system：用户不用关心它干吗的。 下面还将谈到新建pictures文件夹。

如何用Kindle 3浏览照片

Kindle 3中默认并没有图片文件夹可以用来存放图片，但这其实是一个隐藏功能，可以通过在根目录创建pictures文件夹，然后再在pictures文件夹中创建相册文件夹来实现，Kindle把每一个相册文件夹同等地看做一个文档，可以被自动赋予不同分类（Collection，分类的问题后面再写）。在home中，可以直接看到你创建的相册文件夹。

如何用Kindle 3听音乐

alt + space，再按下停止，alt + f下一首，在Menu-Experimental里面的Play MP3旁边有play music，播放时显示pause music。

如何用Kindle 3玩游戏

alt + shift + m，可以出现一个踩地雷的小游戏，按G出现五子棋。

如何在Kindle 3里截取屏幕

alt + shift + G，自动截取当前屏幕，以screen_shot-xxxxx.gif为文件名自动保存在documents文件夹下。

如何在Kindle 3里使用快捷键

请浏览Amazon Kindle 3 键盘快捷键第一版 http://iapp.tk/2010/11/kindle-shortcuts-1ed/ 或者，直接阅读guide，每个kindle里都自带，不用复制了。

ps.对比过很多的东西，初初感觉老外设计的东西功能都比较执着的“单一”，他们为用户需要的每一个细小应用需求都会设计一个产品，不像我们总想把所有的功能都汇集在一个产品中；但那种“单一”中又暗藏很多可发掘的东西，kindle的快捷键可让用户打开并使用它看起来不具有的功能。

如何用Kindle 3读书

可以认为，Kindle 3已经支持了很多格式，但建议你将文字（或者文字为主的混排文档）转换为PDF或者PRC格式，更推荐PRC，因为PRC的文字档相比PDF小多了，这样可以让你的kindle装下一辈子看不完的书，而且打开速度也更快。

如何用Kindle 3完美支持中文 http://bbs.mydoo.cn/thread-7878-1-1.html 字符编码一定是要改的，而且是如下修改，经验证，大小写无所谓。

1，;debugOn 2，~changelocale zh-CN.utf8 3，;debugOff 以上命令，不区分大小写。要输入; ~ 等符号或者数字，按一下sym键就可以出来选择了。第二步也可换为；~changeLocale zh-TW.utf8 命令输入方法：在home界面，按一下回车键（有弯钩箭头的），出来输入框。每次输入一行命令，然后按回车键回车一下，此时屏幕会晃一下，然后又回到了home界面。继续下面的二行。三行命令依次全部输入完毕，回车之后。重启机器。 k3重启方法如下: HOME-menu-Setting-Menu-Restart 如果在完成这个步骤、并将文件转码PRC格式以后，还顽固地出现中文乱码的情况，再考虑机友所介绍的其他办法。

关闭屏保

原生系统下，先按回车键，依次输入：

;debugOn （回车） ~disableScreensaver （回车） ;debugOff （回车

恢复屏保：

;debugOn （回车） ~resumeScreensaver （回车） ;debugOff （回车）

摆脱USB线 WIFI 文件直传 - Kindle 3 usbNetwork 破解流程 http://www.hi-pda.com/forum/viewthread.php?tid=670258&extra=page%3D1&page=1

1. Jailbreak 去网页http://www.mobileread.com/forums/showthread.php?t=88004 下载 kindle-jailbreak-0.10.N.zip。解压后，目录下有一堆文件，是对应不同的 Kindle 版本的。[HOME] → [MENU] > Settings 下可以看到序列号和MAC地址。 k3g = K3 3G US (序列号开头是B006) k3w = K3 WiFi (序列号开头是B008) k3gb = K3 3G UK (序列号开头是B00A) 俺的k3 3g是B006开头的，版本为Kindle 3.3，所以使用 update_jailbreak_0.10.N_k3g-3.2.1_install.bin 文件。插上 K3 的 usb 线，把对应文件拷贝到 K3 的根目录下。在计算机上停止 K3 usb连接，然后在 K3 上进行下列操作 [HOME] → [MENU] > Settings → [MENU] > Update Your Kindle。在升级过程中会显示有错误，错误码是U004，没有关系，这是正常的。K3会自动重启动。

2. 安装 usbNetwork 去网页http://www.mobileread.com/forums/showthread.php?t=88004 下载 kindle-usbnetwork-0.35.N.zip。解压后，像上一步一样选择升级文件。B006的机器使用 update_usbnetwork_0.35.N_k3g_install.bin(不要象我第一次误用了update_usbnetwork_0.35.N_k3g_uninstall.bin而无果)。同样插上usb线，拷贝到根目录，断开usb，然后升级。这次会显示升级成功，然后自动重启。

3. 修改 usbNetwork 配置 重启完毕之后，再次连接USB到计算机。K3 的根目录下会多出一个目录 usbnet。进入该目录，将 DISABLED_auto 文件名改为auto。进入etc子目录，用文本编辑器，如记事本(Notepad)，来修改config文件。这是UNIX格式的文本文件，不要修改成DOS格式。只要把两处false改成true，保存。 K3_WIFI=“true” K3_WIFI_SSHD_ONLY=“true”

4. 制作密钥 去http://www.chiark.greenend.org.uk/~sgtatham/putty/download.html 下载 puttygen.exe。运行该软件，点击 Generate，然后根据提示随机移动鼠标来产生随机密钥。搞完之后，要把 Public key for pasting into OpenSSH authorized_keys file 里面生成的文本粘贴到一个空白的文本文件，应该是巨长的一行。然后保存到 K3 的 usbnet/etc 目录下，文件名是 authorized_keys，这个是公钥。然后点击软件的 save private key 按钮，取个名字，保存到本地硬盘，这个是私钥，步骤6需要使用这个文件。需要的话，可以在Key Passpharse填写一个密码来保护私钥文件。公钥私钥是配对使用的。断开USB连接，[HOME] → [MENU] > Settings → [MENU] > Restart，重启K3。

5. 显示 K3 的 IP 地址 先开启无线，显示 WiFi 图标后，[HOME] → [MENU] > Settings，然后输入 alt+u, alt+q, alt+q。这就是著名的 711 页面。在页面上半部分有 MAC 地址，下半部分显示有 IP Address。一般无线路由器上的DHCP可以根据MAC地址保留IP地址，在路由器里设置一下之后，IP 地址就不会再改变了。

6. SCP 客户端 下载http://winscp.net/download/winscp429.zip。解压缩，软后运行winscp。点击 New 按钮。Host Name 填写上一步得到的 IP 地址。User Name 填 root。Private Key File 选择步骤4保存的私钥文件。Protocol 选择 SCP。左侧点击 SCP/Shell, 然后将 Lookup User Groups 前的勾去除。左侧点击Directories，右侧在Remote Directory中填写/mnt/us。最后点击 Save 按钮保存。将来再次使用的时候，选中保存的配置，直接 login 就可以了。运行之后，左侧是计算机的文件夹。右侧是 K3 文件夹。

如果需要显示中文文件名，经测试，winscp 的 Protocol 选择 SFTP 而不是 SCP。点左侧Environment，右侧UTF-8 encoding for filenames 选择 On 即可。SFTP 的传送速度比 SCP 稍慢。

注意：/mnt/us = USB 连接时 K3 的根目录。拷贝东西到 /mnt/us 下的各个子目录即可。

7. Kindle 文件库的刷新 拷贝到 /mnt/us/documents 下的文件 K3 不会自动检测到，所以需要一个刷新机制，否则每次传完之后需要重启。 用SCP连上K3，进入usbnet目录，按鼠标右键 New → File，文本中粘贴 dbus-send –system /default com.lab126.powerd.resuming int32:1，加一个回车，然后保存。随便取个名字，比如 refresh_kindle。以后每次传完文件，回到usbnet目录，右键点击这个文件，选择 Custom Commands → Execute. 这样不用重启，K3 也能自动更新文件库。

8. SSH 客户端 想要用命令行登入 K3 进行危险操作的同学，可以http://www.chiark.greenend.org.uk/~sgtatham/putty/download.html 下载 putty.exe。用同样的私钥建立SSH连接即可。

去广告，换屏保

所有从WIFI上的下载回来的广告存放在子目录:kindle\system\.assets\中，先删除.assets子目录, 再在system目录下新建一个普通文件,文件名就是.assets, 这样系统就无法创建同名的子目录,于是无法保存广告了。

然后来看自制屏保，在kindle系统opt/amazon/screen_saver下，保存amazon原生系统的屏保图片，包括适用于k3的600×800图片和适用于DX的824×1200图片。

在kindle中，不像PC那样支持用户随手换屏保图片，那么就安装ScreenSaver hack吧。

这个链接是万能的，http://www.mobileread.com/forums/showthread.php?t=88004

1）下载kindle-ss-0.20.N.zip（别纠缠于版本号，随时有更新的）；

2）解压kindle-ss-0.20.N.zip后拷贝update_ss_0.20.N_k3w_install.bin进kindle根目录；

3）Update your Kindle，重启；

4）Kindle根目录下会多出一个目录linkss，里面有个子目录screensavers，用来存放屏保图片；

为了让自制屏幕在kindle里面表现良好，请将图片制作成600×800的灰度文件，格式最好是png。

至此OK了吗？没有，由于破解不完美，重启kindle后发现广告是没了，但出现一个提示你连接网络下载广告的图片，很难看，去之，但这个过程就比较复杂了。我们需要一个牛逼的usbnetwork。

要去掉广告残留图片我们得删个文件，目录/opt/amazon/screen_saver/adunits/600×800，进入目录我们就可以看到那熟悉又厌烦的图片了，删之？提示错误，仔细一瞧发现是没有权限，得输入命令提取权限，按Ctrl+T，输入mntroot rw提取权限，然后输入rm -r /opt/amazon/screen_saver/adunits/600×800，我们再进入就可以看到，600×800这个文件夹已经没有了

到目前可能还有一个问题，屏保图片不换，只有重启才能换一次，这不浪费了辛苦制作的那么多图片了吗？有一个方法是新建个空文件random到linkss下，我试过后发现并不是每次都更换，于是牵绊到这个问题上了，不过，现在你手上的k3已经是非广告版的了，花这么大力气，也算是值了吧。

补充：以上步骤完成后会在屏幕底部留一个空白条，同样输入命令可以解决，在使用winSCP连接，后按Ctrl+T打开terminal依次输入以下各条命令并回车：

/etc/init.d/framework stop mv /var/local/adunits /var/local/adunits.bkp touch /var/local/adunits /etc/init.d/framework start

等等kindle重启后发现，目录布满整页，完美去广告完成。

Kindle3 中文拼音输入法 http://bbs.mydoo.cn/thread-21550-1-1.html

越狱成功之后，根目录下会多出一个linkjail文件夹，里面保存的是越狱程序的所需文件和配置信息，要想安装输入法，必须对设置进行一下修改。我们所需要的修改是其etc目录下的whitelist，也就是白名单，需要在whitelist的文件首添加一行需要放行的安装包前缀，而输入法安装包的前缀是：update_pinyin，修改之后保存即可（压缩包内已经添加了修改之后的白名单，可以直接解压覆盖）。

到这里，我们所需要的一切准备工作已经做完了，可以开始安装拼音输入法了，同样，需要选择与自己机型相符的安装包，其中update_pinyin_0.1_*_install.bin是安装文件，update_pinyin_0.1_k3g_uninstall.bin是卸载文件，把匹配的安装包放在USB的根目录下，同样的方法进行Update，耐心的等待上一会儿，输入法就安装成功了，同样，卸载也是选择相匹配的卸载文件放置在根目录进行Update。
