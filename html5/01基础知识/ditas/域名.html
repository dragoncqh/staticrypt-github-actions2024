域名

“域名”通常是指公共的Internet DNS 系统上实现的域名(任何一个使用IP的计算机网络可以使用DNS来实现他自己的私有名称系统)。

一，概述

网络上计算机的IP地址能够被机器直接读取但难以记忆，于是人们通过域名(Domain Name)来标识网络上计算机/计算机组的地址。域名由一串用点分隔的字符组成的，每个点代表一个域。早期的域名必须以点结尾才能解析,如今DNS服务器已经可以自动补上结尾的点。目前域名长度是最多63个字符,也仅限于ASCII字符的一个子集，这限制了其他语言成为域名(基于Punycode码的IDNA系统可以将Unicode字符串映射为有效的DNS字符集，这已经通过了验证并被一些注册机构采纳为一种变通的方法)。另外，在域名中大小写是没有区分的。域名一般不能超过5级，从左到右域的级别越高，高的级域包含低的级域。域名在整个Internet中是唯一的。

二，历史

1983年DNS最早由保罗·莫卡派乔斯（Paul Mockapetris）发明

1992年域名注册刚刚兴起，申请者寥寥无几。

1993年Internet上出现WWW协议，域名开始吃香。

1993年Network Solutions(NSI)公司与美国政府签下5年合同，独家代理.COM、.ORG、.NET三个国际顶级域名注册权。当时的域名总共才7000左右。1994年开始NSI向每个域名收取100美元注册费，两年后每年收取50美元的管理费。1998年初，NSI已注册域名120多万个，其中90%使用“”.COM““后缀，进帐6000多万美元。

1998年1月30日，美国政府商务部通过其网站正式公布了《域名技术管理改进草案（讨论稿）》绿皮书。对域名系统实施民间化和引入竞争机制，并促进国际的参与。提出组建一个民营的非盈利性企业接管美国政府对域名的管理权。

1998年9月30日美国政府终止了它与目前的域名提供商NSI之间的合同。双方的一项现有协议将延期两年至2000年9月30日。根据该协议，NSI将与其他公司一道承接Internet顶级域名的登记工作。NSI和美国商务部国家电信和信息管理局（NTIA）将于1999年3月31开始分阶段启动共享登记系统，至1999年6月1日完全实施。

1998年10月组建ICANN(Internet Corporation for Assigned Names and Numbers) ，一个非盈利的Internet管理组织。它与美国政府签订协议，接管了原先IANA的职责，负责监视与Internet域名和地址有关的政策和协议，而政府则采取不干预政策。

三，域名的DNS解析

域名解析就是域名到IP地址的转换过程。域名的解析工作由DNS服务器完成。

1，DNS(网域名称系统,Domain Name System，有时也简称为域名）：是因特网的一项核心服务，它是可以将域名和IP地址相互映射的一个分布式数据库。DNS 使用TCP和UDP端口 53。例如，www.wikipedia.org作为一个域名就和IP地址130.94.122.199相对应。DNS在我们直接呼叫网站的域名以后就会把www.wikipedia.org转化成130.94.122.199。

2，域名服务器(Domain Name Server，简称Name Server)是装有域名系统的主机,是一种能够实现名字解析(name resolution)的分层结构数据库。“域名”通常是指公共的Internet DNS 系统上实现的域名(任何一个使用IP的计算机网络可以使用DNS来实现他自己的私有名称系统)。

3, 在DNS系统中分不同的组来负责各子系统的名字。系统中的每一层叫做一个域，每个域用一个点分开。DNS 通过允许一个名称服务器把他的一部分名称服务（众所周知的zone）“委托”给子服务器而实现了一种层次结构的名称空间。此外，DNS还提供了一些额外的信息，例如系统别名、联系信息以及哪一个主机正在充当系统组或域的邮件枢纽。DNS系统是基于13个全球范围的“根服务器”，除了当中的3个以外，其他都位于美国。从这13个根服务器开始,余下的Internet DNS 命名空间被委托给其他的DNS服务器，这些服务器提供DNS名称空间中的特定部分。

4，DNS系统是由各式各样的DNS软件所驱动的，包括：BIND (Berkeley Internet Name Domain, 使用最广泛的名称映射namedaemon),DJBDNS (Dan J Bernstein's DNS implementation),MaraDNS, NSD (Name Server Daemon), PowerDNS

5, 域名解析设置:必须需要到注册商的域名管理界面去设置成正确的有效稳定的DNS。该设置可能需要12-72小时才能反映在根服务器上。用户可以将该域名下的网站服务器指向到自己的web server上。同时也可以设置您域名的二级域名。

A, A (Address) 记录是用来指定主机名/域名对应的IP地址记录。

B, TTL值（Time To Live,生存时间)是DNS记录在DNS服务器上缓存时间。

C, MX记录设置：MX记录是邮件路由记录，用户可以将该域名下的邮件服务器指向到自己指定的IP地址上。(大多数SMTP要求MX记录指向一个主机名。因此请先将MX记录指定到一个主机名，再把该主机名指向IP地址。如：先将MX记录设置为mail或者mail.whatis.cn.,再将mail.whatis.cn解析到18.5.1.249)“优先级”仅对MX记录有效，其他类型的记录将忽略优先级。如果没有自己的邮件服务器，可以使用邮件转发。但做了邮件转发后，将删除原来的MX记录。要谨慎。

D, CNAME记录设置：是别名记录。这种记录允许您将多个名字映射到同一台计算机。通常用于同时提供WWW和MAIL服务的计算机。例如，有一台计算机名为“host.abc.com”（A记录）。它同时提供WWW和MAIL服务，为了便于用户访问服务。可以为该计算机设置两个别名（CNAME）：WWW和MAIL。这两个别名的全称就是www.abc.com和“mail.abc.com”。实际上他们都指向“host.abc.com”。

E, 泛解析:是指“主机名”栏中填“*”而做的全部解析。

6, 因各地的Dns服务器存在缓存问题，如果域名的TTL值为3600，即缓存时间为1个小时，那么修改解析后，在全国范围内全面生效的最长时间是1个小时。另外，自己的电脑也会进行DNS缓存。您可以清空DNS缓存，这可能会使您的域名在您的电脑上立即生效。

刷新DNS解析缓存的具体步骤如下：

在开始>运行处输入cmd 再回车。然后在命令提示符下，键入 ipconfig /flushdns。 在开始>运行处输入 pingwww.yourdomain.com -t 可以测试您的域名解析是否生效。

五，域名管理权

ICANN全面管理DNS，它控制了根域，然后把各顶级域委托给各域名注册局（domain name registry）. 如 ccTLDs的注册机构就是相应的国家。ICANN对这些注册机构担任协调顾问角色，但不参与域名分配、注册的具体章程条件等规定。另一方面，gTLDs直接由ICANN管理，这就意味着所有条款、条件都由ICANN和各gTLDs注册机构共同制定。

一个域名的'所有者'可以通过查询whois数据库而被找到；对于大多数gTLD，基本的WHOIS由ICANN维护，而WHOIS的细节则由控制那个域的域注册机构维护。对于超过240个的CCTLDs，当地国全面负责包括该扩展名的WHOIS在内的诸多功能。

当前对于DNS系统的控制方式，常常受到指责。最常被攻击的焦点是垄断企业或准垄断企业对DNS的滥用，例如VeriSign公司，以及对于顶级域名的分配。

也有些人宣称很多DNS服务器软件无法在动态IP分配上很好的工作，尽管这是某些特定实现的失败而非协议本身的问题。

六，域名的使用

1, 域名结构理论上的本义：按原计划，域名结构遵照严格的层次规定，顶级域名描述组织的类型(商业，政府等)，然后层层下推，联合起来描述一个复杂的结构，如分机构，部门，附属机构等域名会是父机构域名的子域名。同样，主机名原先也计划能够和网络上实际物理主机对应，一机一名。

一个处于http://www.example.org的站点经常以省略http://的方式打广告，一般输入“example.org”就能到达该站。如果是.com，有时（根据浏览器的版本和设置对残缺域名的解释方式）只需要输入“example”就能到达example.com。

2，域名的滥用：当WWW开始流行的时候，站长们希望有好记忆的地址，就不顾其是否符合结构要求。因为.com域名是最流行，最易记的域名，即使是非商业机构也经常使用其下的子域名。即使是父公司有了二级域名，各种各样的子公司子机构站点也希望有自己独立的二级域名。

3，域名争议：域名的流行造成的域名的滥用，引起了拥有商标权的实体公司的争议，这被称为cybersquatting。比如某人创建一个和某商标类似的域名，以图从到该站点的流量中获利。为了打击这种行为，各种各样的法律和政策纷纷出台，试图使被滥用的域名被强制过户。但有时也会矫枉过正，一些过分热心的公司也滥用反域名劫持活动( domain hijacking )，提起针对某些域名合法拥有者的争议。比如域名在某些上下文中含有和商标名同样的文字，或者他们以其自己的言论自由权利使用诸如fan,protest等上下文。

4，通用性域名的不规范使用问题：由于域名是先到先得，对某些通用产品或者服务的域名也是如此，这样会造成域名的不规范和误导。如艺术节literary.org。一些批评家提出应该保留这些地区性和总概括性的艺术节组织的域名使用，以作为改进。

5，域名的另类用法：由于单字母.com域名的稀缺，许多非传统的域名，域名分割(domain hacks,) 正变得流行。两个流行的分割式域名是del.icio.us 和 blo.gs,它们的原拼写分别是delicious和blogs。非传统的域名也经常用来创建非传统的邮件地址。比如把james拼成j@m.es 或j@mes.com。

6，域名防止误解：注册域名时要小心：DNS是大小写不敏感的。而现代词汇的趋势是CamelCase模式的连写，这样在转换成小写后会造成误解。Who Represents是一个作者和代理人的数据库，使用域名 whorepresents.com，而whore是妓女或青楼，presents是礼物。临床医学家therapists网络使用域名 therapistfinder.com ，而rapist是强奸者。笔具定制公司 Pen Island 注册了 penisland.com; 程序员网站Experts Exchange曾长年使用 expertsexchange.com。在这种情况下，正确的词汇可以通过连字符来区分出来。比如 expertsexchange.com 可改名为 experts-exchange.com。DNS是大小写不敏感的，所以上述站点也可以在广告中通过进一步的大小写以区分意思，如TherapistFinder.com。

七，免费域名

一些公司提供低成本，亏本甚至完全免费的域名注册，但会要求遵守各种各样的注册模式以补偿他们的成本。通常是要求该域名必须以框架或者门户等方式绑定到他们的主机，并在用户页面上放置广告，以其收入抵偿成本。当新DNS出现时，域名注册通常是免费的。一个域名的拥有者也通常可以赠送或者销售无限的子域名。

2, 域名的价值：域名(尤其是和生意business, 投机赌博gambling, 色情pornography, 和其它有利可图的领域等相关的)需求量巨大。域名对网络品牌建设，广告，搜索引擎优化等有重要影响。高价域名的一个原因是即使不做广告或者市场推广，它们也能吸引那些寻找产品或者服务的直接输入通用名的客户，而且，诸如 movies.com 或者 Books.com 之类的域名非常容易让潜在客户们记住，这增加了他们成为回头客或者固定客户的可能性。

3，域名交易的市场规模：域名的普及带来了通用域名转售市场近十年来的兴旺。虽然现在的域名市场无法和.com全盛时期相提并论。但它仍然保持强势且正再次稳定增长。每年的域名转售交易额数以千万美元计。每年都有大匹域名过期失效和删除。平均每天有25,000个域名失效被删除。

九，中文域名

中文域名(中文网址)是指相对于由英文字母组成的英文网址（如www.cnnic.cn）而言，由中文/汉字组成的互联网网址，也被称为中文域名。比如“中国工商银行”的英文网址为“www.icbc.com.cn”，中文网址为“工行”，即在浏览器的地址栏里输入中文/汉字“工行”二字即可直达中国工商银行的网站。

相对于英文域名而言，中文网址的现状比较混乱。例如，北京3721科技有限公司在推广其网络实名产品，而中国互联网信息中心推出的服务叫通用网址。两者都允许在浏览器的地址栏输入中文域名而直达相关网站，但是互不兼容。在国际上，也有其他公司提出中文网址的地解决方案，甚至还曾有关于“www.国务院.com被某外国公司注册”的报道。近期，关于中文网址的法律纠纷不断，一些法学学者和互联网业内人士认为，应该由中国政府、信息产业部出面制定相关法律法规。
